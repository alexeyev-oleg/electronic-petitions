# Getting Started

## What This App Is
The Resident App is the mobile entry point for:
- petitions
- complaints
- municipal violation reports

This version is an early beta focused on testing the user experience and the main flows.

## Beta Notes
- This beta uses simplified registration.
- Sign-in currently works with `email + password`.
- Secure identity verification will be added later.
- Some actions may be test-only until the secure backend flow is enabled.

## Supported Languages
- English
- Hebrew
- Russian
- Arabic

## First Launch
1. Open the app and wait for the splash screen.
2. Swipe through onboarding and choose your language on the first slide.
3. Tap `Get started`.
4. Create an account or sign in.
5. Explore the main sections:
   - Petitions
   - Complaints
   - Report a Violation
   - Profile
   - Inbox

On later launches, onboarding is skipped and the app goes directly to sign-in.

## Help and Support
From `Profile`, open `Help and support` to access:
- FAQ
- About the app

## Main Sections
### Petitions
Use this area to browse and later create civic petitions.

### Complaints
Use this area to report city problems and follow their status.

### Report a Violation
Use this area to submit a possible municipal enforcement report with media and location information.

### Profile
Use this area to manage your account, language, notification preferences, and help content.

## What to Expect in Beta
- Some screens may use mock data saved locally on your device between app restarts.
- Some actions may be placeholders while backend integration is being completed.
- In Profile, you can enable `Simulate list load error` to test retry states on list screens.
- Design and user flow may still change based on testing feedback.
